## Issues Found & Resolution

### ✅ CRITICAL - FIXED: Invalid Database Query in `/api/company/data`

**Issue**: The endpoint was querying a non-existent `user_type` column in the `profiles` table, causing 400 errors:
```
Error: column profiles.user_type does not exist
Code: 42703
```

**Root Cause**: 
- The `profiles` table uses a `role` column (admin, dispatcher), not `user_type`
- Drivers and subcontractors are stored in separate `conductores` and `subcontractores` tables
- The query was attempting to filter by a column that doesn't exist

**Fix Applied**:
- Removed the broken Supabase queries
- Now using the correct static data imports from `all-drivers.ts` and `all-subcontractors.ts`
- Endpoint returns 200 OK with proper data

**File Modified**: `app/api/company/data/route.ts`
